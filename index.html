Proper SSL

Proper SSL is a repository of examples of how to properly setup SSL connections as both a client and server.

About

The idea for Proper SSL came about when looking for good examples for setting up SSL on nginx (specifically how to set it up for PFS using ECDHE). The idea was then expanded to include other client/server setups which should be correctly using SSL such as database clients and servers.

Format

Each repository contains a specific self-contained example of how to properly setup and use an SSL connection. The objective isn't to create the most extensible SSL examples, it's simply to have a cheat sheet to look at for the right way of doing things. Whenever possible external dependencies should be minimized.

Each example should preferably be runnable. Given the client/server nature of most of these examples the preferred approach is to define the test server using Vagrant so that new users can try things out with a simple vagrant up.

Any additional steps needed to run the example should be in the example's README file.

Repo Naming Conventions - Clients Connecting To Servers

Examples of clients securely connecting to servers should be named in the following format:

[language]-[interface/framework]-[target]
  • language is the programming language used (eg java, ruby, python).
  • interface/framework is the framework used (eg jdbc, sqlalchemy, datamapper, dbi).
  • target is the target that is being connected to. This will usually be an external server (eg postgresql, mysql)

Unless there's a good reason for it don't add "ssl" to the name of the repo.

Repo Naming Conventions - Server Configurations

Examples of server configurations should be named in the following format:

[server-name]-[description]
  • server-name is the name of the server program (eg nginx, apache2, mysql)
  • description is a short description about what's being configured (eg pfs-ecdhe)
